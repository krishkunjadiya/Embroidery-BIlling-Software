# Security Policy

## Supported Versions

Security updates are provided for:

- `main` branch (latest)

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report responsibly by:

1. Opening a private security advisory on GitHub, or
2. Contacting the maintainers directly.

Include as much detail as possible:

- Affected component or file
- Reproduction steps
- Potential impact
- Suggested mitigation (if known)

We will acknowledge reports as quickly as possible and work on a fix timeline
based on severity.
